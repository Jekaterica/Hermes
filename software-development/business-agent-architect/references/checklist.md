# Чеклист приёмки агента

## Функциональность
- [ ] Агент отвечает строго на заданный вопрос, без лишнего
- [ ] Все утверждённые сценарии работают без ошибок
- [ ] При отсутствии данных — «нет информации» + эскалация
- [ ] При нештатной ситуации — эскалация человеку
- [ ] Агент не фантазирует (проверено на запросе без данных в БД)

## Безопасность
- [ ] Агент не выполняет опасные действия без подтверждения
- [ ] State machine не допускает некорректных переходов
- [ ] Нет тупиковых состояний
- [ ] Конфиги и токены не в коде, а в .env

## Производительность
- [ ] Время ответа < 2 секунд
- [ ] Агент не зависает при высокой нагрузке

## Надёжность
- [ ] После перезагрузки сервера состояние восстанавливается из БД
- [ ] Логирование всех действий включено
- [ ] Fallback на человека работает

## Юзабилити (для клиента)
- [ ] Клиент не может запутаться в диалоге
- [ ] Любой запрос обрабатывается (даже если ответ «нет данных»)
- [ ] Есть команда /help и /admin

## Юзабилити (для владельца)
- [ ] Владелец может смотреть логи
- [ ] Владелец может временно отключить агента
- [ ] Владелец может менять данные (прайс, часы) без твоего участия

## Подход
- [ ] Итеративный: начали с уровня 1-2, без «экосистемы на вырост»
- [ ] Сложность растёт под конкретного клиента, а не из общих соображений
