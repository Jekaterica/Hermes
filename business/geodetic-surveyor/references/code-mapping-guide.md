# Настройка code_mapping.yaml

## Проблема

У каждого геодезиста своя система кодирования точек:
- «Д1» может означать «дерево лиственное» или «дорога 1 категории»
- «ЗД» — «здание»

Система не может угадать. Решение: геодезист один раз настраивает YAML-файл.

## Формат

```yaml
# mapping кодов точек → слои, цвет, условный знак
# геодезист настраивает один раз под свои коды

ЗД:
  layer: ЗДАНИЯ
  color: "#FF0000"  # красный
  block: BUILDING
  closed: true      # true = замкнутый контур

ДОР:
  layer: ДОРОГИ
  color: "#404040"
  block: none
  closed: false

ДР:
  layer: ДЕРЕВЬЯ
  color: "#008000"
  block: TREE
  closed: false

# Фильтры
filters:
  z_threshold_mm: 500           # выбросы по Z от медианы
  duplicate_xy_tolerance_mm: 10 # дубликаты по XY
```

## Цвета (HEX → ACI AutoCAD)

| HEX | ACI | Цвет |
|---|---|---|
| #FF0000 | 1 | Красный |
| #FFFF00 | 2 | Жёлтый |
| #00FF00 | 3 | Зелёный |
| #00FFFF | 4 | Голубой |
| #0000FF | 5 | Синий |
| #FF00FF | 6 | Фиолетовый |
| #404040 | 8 | Тёмно-серый |
| #FF8C00 | 30 | Оранжевый |
| #8B4513 | 32 | Коричневый |
| #008000 | 130 | Тёмно-зелёный |

## Pitfalls

- Не использовать русские названия блоков (AutoCAD плохо их понимает). Транслит или английский.
- Коды чувствительны к регистру: «ЗД» ≠ «зд»
- Если код не найден в маппинге — точка падает в слой ОБЩИЕ с цветом 7 (белый)
